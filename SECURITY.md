# Security Policy

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security reports.

Send details to **codeekage@users.noreply.github.com** (GitHub-routed) or
open a private security advisory via the **Security** tab on this
repository (`Security` → `Advisories` → `Report a vulnerability`).

Include, where possible:

- A clear description of the issue and its impact.
- Repro steps or a minimal test case.
- The commit SHA / version you reproduced against.

You will receive an acknowledgement within a few days. Fixes for
confirmed issues will be coordinated privately before public disclosure.

## Supported Versions

This project is pre-1.0. Only the `main` branch is supported; please
reproduce against the latest commit before reporting.
